A Semantic accessibility example

Article heading

This page contains some of the most used elements optimized for accessibility

A common misconception is that Accessibility is only for Visualy handicapped users. Although this group normaly provides the biggest technical challenges the other handicapped users should not be forgotten. The impaired users can be split into four groups:

  • Visual.
    This can be non-sighted users, users with low-vision, users with obstructed vision, or even simply your aging parents.
    • Font resize friendly text.
    • Enough contrast throughout your site (minimum of 4.5:1 for text and 3:1 for large text)
    • Design with color blindness in mind.
    • Usage of correct html attributes (e.g. alt text on images)
    • Usage of Aria attributes when needed (For example custom elements).
  • Auditory.
    The web is a visual medium, but captions and fallbacks for sound-necessary media need to be considered for hearing impaired users.
    • Captions for video's.
    • Transcripts for audio files.
  • Motor.
    People with motor impairments typically use a wide range of assistive technology from specialized keyboards, to eye trackers, to using single buttons (a device known as a “switch”) to navigate their computers.
    • Large click area's for interactive elements (minimum button size).
    • Layout can be navigated though with only an keyboard.
  • Cognitive.
    Relates to the ease processing of information. (Dyslexia, Autism, Global developmental delay)
    • Usage of readable fonts.
    • Easy to understand text.

Use for -> id to link a label to an input
<label for="name">Name:</label>
<input id="name" type="text" name="textfield">
Or use an title to create an accessible input
<span>Last name:</span>
<input title="Last name" type="text" name="textfield">
Last name:
But do not combine both, it will create redundant speech!
<label for="oopsname">REDUNDANT LABEL</label>
<input id="oopsname" type="text" name="textfield" title="REDUNDANT TITLE!">
You can use an aria-label to overwrite a label tied to an input. This can be usefull for when the label is way to long for a screenreader to read and could be shortened
<label for="name2">This input contains an aria label and will not read what is written here!!</label>
<input id="name2" type="text" name="textfield" aria-label="Aria overwritten label">

<label for="name3">
An other input.
<b aria-hidden="true">
This is a required field. (This bold text is hidden for screen readers)
</b>
</label>
<input id="name3" type="text" aria-required="true" required name="textfield">
